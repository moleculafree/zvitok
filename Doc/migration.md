# Міграція

Процедура міграції призначена для внесення змін в бази даних програми через сценарії (файли скриптів). Окрім офіційних файлів міграції що поставляються разом з програмою, користувач може самостійно створювати та додавати скрипти в поточну версію міграції.    

## Версії та скрипти міграції

Процедура міграції виконується з файлами поточного профілю під час запуску програми або зміни організації в програмі.
Процедура складається з покрокового обходу тек з номерами версії міграції та одноразового виконання вмісту файлів в яких описуються дії в форматі SQL.

Файли міграції які були вдало виконані та зафіксовані в журналі міграції більше не беруть участь в цій версії міграції.
Після завершення міграції в файл бази записується номер версії міграції. 


### Номер версії міграції

Назва теки завдає номер версії міграції. Номер версії складається **виключно з цифр**, початкова версія починається з 000.   

Приклад:
000 - перша версія міграції  
001 - наступна версія  
002 - наступна версія  
...


### Скрипти міграції

Кількість скриптів міграції не обмежено, назва файу міграції повинна починатись з цифри (крок міграції), символа підкреслення та короткого опису.
Приклад:
```
1_create_table.sql
2_update_index.sql
...
9_fix_zvit_number.sql
```

За потреби, в теці поточної версії міграції можете розміщувати свої скрипти дотримуючись правила формування назви файлу.


Тека з файлами для міграції профіля:
\Data\System\Schema\Migration\Profile

Тека з файлами для міграції журнала подій:
\Data\System\Schema\Migration\Journal

### Архівні копії файлів баз

Тека в яку зберігаються копії файлів баз даних перед виконанням кроку міграції:
**\Tmp\Migrated\**

Формат назви файла:
**<початкова_назва_файла>_датачас**

Приклад:
1234567.db_202607171212
1234567.journal_202607171012
